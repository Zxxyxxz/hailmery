## What APIRE is — the canonical one-liner

**APIRE.IO is an AI API Security and Governance Platform** that operates as a **transparent security proxy** between organizational applications and AI providers. It delivers real-time inspection, threat detection, content moderation, and data protection on every prompt and response, under a **Zero-Retention Architecture (ZRA)** — all data processing happens in volatile RAM and is irrevocably erased upon completion. Integration is zero-touch and zero code change: a single endpoint URL swap.

## Business description (long-form, exact phrasing from Brand Context Document)

> APIRE is the purpose-built AI security platform that sits between your app and AI providers. We block 27+ AI-specific threats in real time, prevent PII and proprietary data leakage with on-the-fly masking, and enforce granular policies across OpenAI, Anthropic, Gemini, and more. Prove NIS2/EU AI Act compliance with live dashboards, content safety, and cost analytics—all in one platform. Deploy in 5 minutes. Zero code changes. Keep shipping, safely. The APIRE product suite extends the same 5-layer protection across every AI surface: APIRE Security Proxy (the core), APIRE Secure Chat (sanctioned ChatGPT-style interface for employees, AD/LDAP integrated), APIRE Browser Agent (browser-level protection for ChatGPT, Claude, Gemini and any web AI tool), and APIRE Log Appliance (dedicated AI security event collection with SIEM webhook integration).

## Contradictions to flag

- **"Four-layer" vs "five-layer" architecture:** The Solution Architecture Document v2.1 (Nov 2025) and several older briefings describe a **four-layer defense system** (Layers 1–4). The 2026 Datasheet introduces **Layer 0 — Sentinel — AI-Powered Gateway** in front of those four. **Current canonical framing is FIVE layers** (Sentinel + the four detection shields). Older "four-layer" copy is not wrong about the detection shields, but the headline number is now five.
- **ZRA tagline punctuation:** "Process Everything. Keep Nothing. Protect Always." and "Process Everything. Store Nothing. Protect Always." both appear in source material. Use **"Keep Nothing"** as the preferred form (it appears more often in technical and architectural documents) but treat "Store Nothing" as acceptable in marketing contexts.
- **Endpoint URL:** `app.apire.io` is the canonical proxy endpoint. `app.apire.ai` appears in older Solution Architecture doc copy and the Zero-Retention briefing — treat as the same endpoint; prefer `.io` in new content.
